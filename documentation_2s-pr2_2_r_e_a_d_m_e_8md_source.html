4 ## Introduction
5 
6 S-PR2 is a new and powerful Python based software package that can be used as a kinematic control toolbox for PR2 service robot.
7 The software exploits an analytic Inverse Kinematic solver for PR2 redundant arm (7 DOF)
8 with capability of optimum redundancy resolution.
9 S-PR2 can be used by robot researchers and programmers to generate their desired trajectories in the operational space and execute
10 on a physical robot or in simulation.
11 The package gives PR2 robot, the skills of tracking a user-defined trajectory like writing letters or drawing shapes on the board.
12 
13 S-PR2 provides an efficient and reliable development environment providing optimal and feasible kinematic control
14 functions for PR2 users. S-PR2 has adopted the lastest techniques of the Inverse Kinematic (IK) solution in recent years.
15 
16 ### What is PR2?
17 
18 The PR2 is a wheel-based service robot produced by
19 [Willow-Garage](https://www.willowgarage.com/)
20 with two 7-DOF arms, a tilting head and a sliding joint,
21 adjusting the working height of the arms adding one degree of freedom
22 to the system.
23 
24 <img src="https://github.com/uts-magic-lab/Magiks/blob/master/documentation/s-pr2/figures/PR2_Robot_Willow_Garage_3.jpg" width="700">
25 
26 The navigating platform can move on the floor and rotate around the **z** axis providing three extra degrees of freedom.
27 In free-base mode, eleven degrees of freedom kinematically influence the position or orientation of one gripper.
28 
29 ### Currently Available IK solver (IK Fast)
30 
31 The available [IK solver](http://wiki.ros.org/pr2_kinematics/Tutorials/Tutorial%204) for the PR2,
32 which is a part of Robot Operating System (ROS), is fast but
33 lacks some important advanced features required for generating smooth and reliable trajectories.
34 The existing package:
35 
36 * Only projects a single pose to the joint-space and not a trajectory
37 * Does not provide all the available solutions (corresponding to a certain value of a redundant parameter)
38 * Can not exploit the available redundancy in Degrees of Freedom (DOF) to minimise a desired cost function or to fulfil additional constraints
39 * Cannot give self motion trajectory connecting two points in the solution manifold
40 * Does not return a feasibility set for a redundant parameter to be chosen
41 * Is fully embedded within the ROS system and cannot be easily replaced or migrated to other systems
42 * Its algorithms are only provided in binary and so is not available to inspect or modify
43 
44 ### Why use S-PR2?
45 
46 We introduced a comprehensive kinematic control software package for the PR2 robot,
47 which is based on an analytic inverse kinematic solution and addresses all the aforementioned issues.
48 S-PR2 generates more effective robot motion plans,
49 hence it can be used to produce morereliable robot behaviours.
50 In S-PR2, the method used for redundancy resolution exploits the advantage of the existence of an analytic IK solution for
51 improving an iterative redundancy optimization in terms of reliability, accuracy, and computational cost where
52 closed-form IK equations are used to reduce dimensionality and complexity of the objective function.
53 
54 S-PR2 allows an end user to easily generate desired trajectories in task-space, project them to the joint-space while
55 exploiting DOF redundancy, visualize the trajectories, and finally,
56 execute them on a PR2 robot using simple Python function calls.
57 The typical user does not need any specific knowledge of the underlying robot control system in order to use S-PR2.
58 Expert robot programmers can, however, use the lower level function modules in S-PR2 to write customized IK control systems.
59 Since, S-PR2 is written in pure Python scripting language,
60 the kinematic software is agnostic to the underlying operation platform.
61 
62 In the following sections, we will first describe the system structure and core functional modules in S-PR2, then
63 each subcomponent will be briefly explained in terms of their methods and capabilities.
64 Finally, an example of a practical application of the package is demonstrated.

134 
135 Module [```PR2_arm_kinematics```](http://uts-magic-lab.github.io/Magiks/namespacemagiks_1_1specific__geometries_1_1pr2_1_1pr2__arm__kinematics.html)

137 This module provides methods required for kinematic computations of the PR2 arm.

139 ### Analytic Inverse Kinematic Solver
140 
141 The most important part of the module is an analytic IK solver that can give all the feasible configurations
142 corresponding to a desired EE pose for an arbitrary value of the redundant parameter (up to maximum 8 solutions).
143 The redundant parameter is by default selected as the first arm joint named Shoulder-Pan Joint.
144 If the given task pose is outside the arm’s reachable region or
145 an ill chosen value is used for the redundant parameter,
146 no solution can be found by the solver.
147 Since identifying the reason for failing to find a solution can be difficult,
148 S-PR2 provides a feasibility Permission Set for the redundant parameter and
149 searches within this set to find a value that
150 leads to a solution with respect to the user’s desired criterion,
151 i.e. the distance of the current configuration from the middle of the joint ranges.
152 If the computed permission set is empty, then
153 it is certain that the requested task pose lies outside of the robot workspace in which case no valid solution exists.
154 
155 ### Permission Set
156 
157 In IK problems, the key challenge is to use redundancy to find the optimal joint values within their feasible ranges.
158 Each joint limitation makes part of the solution manifold invalid
159 and imposes a restriction on the feasible domain from which the redundant parameters can be chosen.
160 For the PR2 that has only one redundant parameter in its manipulator,
161 a Permission Set consisting of a union of intervals can be obtained by the intersection of feasibility permission sets
162 imposed on the redundant parameter by the limitations of each joint.
163 In other words, a permission set is a set from which the redundant parameter must be selected otherwise
164 no solution in the feasible range can be found for the desired EE pose.
165 S-PR2 can compute the permission set for a given desired EE pose using
166 [Arithmetic of Intervals](https://en.wikipedia.org/wiki/Interval_arithmetic).
167 
168 ### Optimal Redundancy Resolution
169 
170 A task in the cartesian space requires six kinematic constraints for the endeffector.
171 If the number of free joints (DOF) is greater than the number of constraints required to fulfil a task,
172 the robot is said to be in a Redundant state.
173 In this case, one or some of the joints can be arbitrarily chosen as redundant parameters.
174 It is also possible to consider a user-defined function of joints as a redundant parameter.
175 Selecting an appropriate function for redundant parameters is called parametrization and it depends upon the geometry of the manipulator.
176 Each redundant parameter adds an additional constraint to the system, so the total number of redundant
177 parameters denoted as Degree of Redundancy is computed
178 as *r = n − m* where *n* is the degree of freedom and *m* is the number of main kinematics constraints
179 (*m = 6* for a desired end effector position and orientation).
180 
181 Some redundancy parametrizations have been proposed for specific geometries, like
182 [Lee and Bejczy](http://ieeexplore.ieee.org/xpl/login.jsp?tp=&arnumber=131621&url=http%3A%2F%2Fieeexplore.ieee.org%2Fxpls%2Fabs_all.jsp%3Farnumber%3D131621)
183 who used some of the joints as redundant parameters in a 8 DOF arm or
184 [Shimizu et.al.] (http://ieeexplore.ieee.org/xpl/login.jsp?tp=&arnumber=4631505&url=http%3A%2F%2Fieeexplore.ieee.org%2Fxpls%2Fabs_all.jsp%3Farnumber%3D4631505),
185 who used a user defined angle as the redundant parameter in a 7 DOF arm.
186 The analytic solution of IK in redundant systems is to express joint positions as
187 closed-form functions of redundant parameters, and
188 the problem of Optimal Redundancy Resolution in analytic IK
189 is to find a feasible value for the redundant parameter(s) to
190 minimize a desired cost function subject to some additional constraints.
191 The method used for redundancy resolution in fixed-base mode is similar to Shimizu’s work but with a different parametrization.
192 The analytic IK method used is also different as their formulations could not be used for the PR2 arm due to a slight difference in the [Denavit-Hartenberg (DH) parameters](http://wiki.ros.org/pr2_calibration_estimation) of the PR2 arms.

194 ### Smooth Trajectory Projection
195 
196 The arm kinematic module can project a trajectory from task-space into the joint-space.
197 Trajectories are instances of module Trajectory consisting of segments
198 established from a number of key points containing positions, velocities and accelerations.
199 Each key point can apply constraint(s) to the trajectory.
200 Trajectory points are interpolated from a fitted spline in form of a polynomial or Fourier series.
201 The user can visualize both joint-space and task-space trajectories before executing them on a physical robot.
202 Orientation-Trajectory is a module inherited from Trajectory supported by various methods for smooth orientation trajectory interpolation. Given the key orientations and corresponding angular velocities and accelerations,
203 the interpolated orientation can be computed at any phase value using methods like [SLERP](https://en.wikipedia.org/wiki/Slerp)
204 with various conventions like quaternions or rotation matrix.
205 The module also supports polynomial interpolation using one of the
206 [vectorial](http://link.springer.com/article/10.1023%2FA%3A1024265401576)
207  non-redundant representations of three-dimensional rotation.

210 ### Self Motion Generation
211 
212 In a redundant manipulator, if the EE is fixed in some pose,
213 the joints can still change in the solution manifold
214 to generate a motion that does not influence the end effector pose.
215 This is known as the *self Motion*.
216 Different values of the joints in the solution manifold can be achieved by changing the redundant parameters.
217 The arm kinematic module in S-PR2 is able to generate a smooth internal motion
218 within the solution manifold, connecting two different IK solution points in the joint-space.
219 This enables the robot to change itself into a better configuration while the end effector pose is fixed.

221 ## PR2 Kinematics Module
222 
223 The
224 [```PR2_Kinematics```](http://uts-magic-lab.github.io/Magiks/namespacemagiks_1_1specific__geometries_1_1pr2_1_1pr2__kinematics.html)
225 module is a comprehensive toolbox for all kinematic computations of the PR2.
226 It exploits the arm local IK solver to compute the configuration in terms of
227 eleven joints (degrees of freedom) given the desired end effector pose in the global coordinate system.
228 This module also generates optimal trajectories for each of the eleven degrees
229 of freedom corresponding to a desired task.
230 
231 ### Redundancy Resolution in Free-Base Mode
232 
233 In free-base mode, the redundancy resolution technique
234 used for the arm is extended to five redundant parameters for
235 the entire robot. In free-base mode, the position, orientation
236 and height of the robot trunk are computed so that a desired
237 cost function is minimized. This is another useful feature that
238 can not be found in any of the existing packages for PR2.
239 With the closed form arm IK equations,
240 the redundancy optimization problem is transferred from the eleven dimensional joint-space
241 into a five dimensional solution manifold spanned by the redundant parameters.
242 In this method, gradients of the objective function with respect to redundant parameters are
243 computed to find an optimum direction for the change of redundant parameters and hence the joint values.

245 ## Application Example
246 
247 Writing and drawing are complicated tasks that require
248 high accuracy and an understanding of the environment. We
249 used S-PR2 to create a system that enables PR2 to write
250 letters or draw any desired shape on the board: Writer-PR2.
251 Raw trajectories are given by an external device such as an
252 iPad tablet or loaded from a file containing some predefined
253 trajectories (for example English alphabets), then mapped
254 into the desired size, location, and orientation. They are
255 then projected into a feasible joint-space trajectory according
256 to the current robot state and configuration. The shapes or
257 letters are drawn/written by any of the arms with desired size
258 and speed.
