1 # Installing MAGIKS
2 
3 MAGIKS is a set of scripts for python developers. It can be easily installed by checking out the spurce code from the github repository. Make a directory anywhere in which you want to copy the scripts. This will be the root directory of MAGIKS. Then go to that directory and type:
4 
5 ```
6 git clone https://github.com/uts-magic-lab/Magiks.git
7 ```
8 
9 Before using MAGIKS, you need to install some dependant packages:
10 
11 ##Prerequisites
12 
13 Currently, Magiks is dependant on the following Python library packages
14 
15 * crlibm (required by pyinterval).
16 * pyinterval.
17 * matplotlib.
18 * numpy.
19 * sympy.
20 * cgkit.
21 * PyRide (If you want to use pyride ROS interface for S-PR2)
22 
23 The following sections give detailed instructions on how these required libraries can be installed on your system. **NOTE** that we assume your target machine use a recent Ubuntu Linux system.
24 
25 ### Install numpy,sympy and other required dependancies
26 First, we install required packages from the standard Ubuntu distribution repository:
27 
28 1. ```sudo apt-get update``` (make sure we have latest package listing).
29 2. ```sudo apt-get install python-pip python-pygame python-matplotlib python-numpy python-sympy scons```.
30 
31 ### Compile and install crlibm library
32 There is no standard package for crlibm library on Ubuntu/Debian system. The
33 source code can be downloaded from [here](http://lipforge.ens-lyon.fr/frs/download.php/162/crlibm-1.0beta4.tar.gz). Use the following commands to compile and install the library from a terminal:
34 
35 ```
36 tar zxf crlibm-1.0beta4.tar.gz
37 cd crlibm-1.0beta4
38 ./configure CPPFLAGS=-fPIC
39 make
40 sudo make install
41 ```
42 
43 The crlibm library should be installed under this directory:
44 ```
45 /usr/local/lib
46 ```
47 
48 ### Install pyinterval package
49 ```
50 sudo pip install pyinterval
51 ```
52 
53 ### Compile and install cgkit
54 cgkit also requires custom installation. The source code can be downloaded from [here](http://liquidtelecom.dl.sourceforge.net/project/cgkit/cgkit/cgkit-2.0.0/cgkit-2.0.0-py3k.tar.gz). Similar to crlibm, use the following commands to install cgkit library from a terminal:
55 
56 ```
57 tar zxf cgkit-2.0.0-py3k.tar.gz
58 cd cgkit-2.0.0/supportlib
59 scons
60 cd ..
61 sudo python setup.py install
62 ```
63 
64 ### Install PyRide package
65 
66 PyRide is required if you want to control the physical PR2 robot or PR2 in simulation via S-PR2 package under MAGIKS.
67 It is a ROS interface by which you can communicate with ROS and can synchronize the configuration of the PR2 object with the real robot.
68 You can also run a trajectory or implement a real-time trajectory control on PR2.
69 You will need PyRide if you work with any of these packages:
70 
71 * [```PyRide_Synchronizer```](http://uts-magic-lab.github.io/Magiks/namespacemagiks_1_1specific__geometries_1_1pr2_1_1pyride__synchronizer.html)
72 * [```pyride_interpreter```](http://uts-magic-lab.github.io/Magiks/namespacemagiks_1_1specific__geometries_1_1pr2_1_1pyride__interpreter.html)
73 * [```Skilled_PR2```](http://uts-magic-lab.github.io/Magiks/namespacemagiks_1_1specific__geometries_1_1pr2_1_1skilled__pr2.html)
74 * [```Writer-PR2()```](http://uts-magic-lab.github.io/Magiks/namespacemagiks_1_1specific__geometries_1_1pr2_1_1writer__pr2.html)
75 
76 To install pyride, follow the instructions from [here](https://github.com/uts-magic-lab/pyride_pr2/blob/master/README.md).
